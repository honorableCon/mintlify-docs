## Démarrer en 4 étapes

Ce guide vous permet de faire votre première intégration Kiriku (auth, clé API, extraction), puis de vérifier votre solde et la facturation.

### Étape 1 : Authentification (JWT)

Récupérez un token via l’endpoint de login:

```bash
curl -X POST https://api.kiriku.app/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "password": "your_password"
  }'
```

Vous obtenez un `accessToken` (JWT) à envoyer dans:

```txt
Authorization: Bearer <JWT>
```

### Étape 2 : Créer une clé API

Une clé API est pratique pour l’intégration serveur (sans session). Créez-la avec votre JWT:

```bash
curl -X POST https://api.kiriku.app/v1/api-keys \
  -H "Authorization: Bearer <JWT>" \
  -H "Content-Type: application/json" \
  -d '{ "name": "backend-prod" }'
```

La réponse contient une clé que vous utiliserez ensuite comme Bearer token (API key).

### Étape 3 : Faire une extraction (multipart)

Endpoint: `POST /v1/extract`

Exemple minimal (fichier recto seulement):

```bash
curl -X POST https://api.kiriku.app/v1/extract \
  -H "Authorization: Bearer <API_KEY>" \
  -F "file=@/path/to/document.jpg" \
  -F "documentType=cni_senegal" \
  -F 'options={"returnRawText":false}'
```

### Étape 4 : Billing (crédits, factures, abonnements)

Les écrans et endpoints utiles:

- Solde crédits et plan: `GET /v1/users/me`
- Factures: `GET /v1/billing/invoices`
- Achat pack: `POST /v1/billing/checkout`
- Abonnement interne + facture initiale: `POST /v1/billing/subscribe`
- Annulation (pas de downgrade immédiat): `POST /v1/billing/subscription/cancel`

Pour le détail, voir la [référence Billing](/api-reference/introduction).
