## Base URL

Production : `https://api.kiriku.app`

## Authentification

Kiriku accepte :

- `Authorization: Bearer <JWT>`
- `X-API-Key: sk_live_...`

Les endpoints d’administration exigent le rôle `admin`.

## Format des erreurs

```json
{
  "statusCode": 400,
  "message": "Invalid request",
  "error": "Bad Request"
}
```
