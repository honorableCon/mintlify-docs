---
title: "Gestion des erreurs"
description: "Codes HTTP et structure des erreurs"
---

Kiriku retourne des codes HTTP standards. Les erreurs NestJS suivent ce format :

```json
{
  "statusCode": 400,
  "message": "Invalid documentType",
  "error": "Bad Request"
}
```

## Codes fréquents

| Code | Signification |
| :--- | :--- |
| 400 | Requête invalide |
| 401 | Non authentifié |
| 403 | Accès refusé |
| 404 | Ressource introuvable |
| 409 | Conflit (ex: clé déjà existante) |
| 429 | Trop de requêtes |
| 500 | Erreur serveur |

## Erreur 429 (rate limit)

Exemple :

```json
{
  "statusCode": 429,
  "message": "Rate limit exceeded (60/min)",
  "error": "Too Many Requests"
}
```
