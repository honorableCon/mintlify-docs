title: "Authentication"

---

The Kiriku API uses API keys to authenticate requests. You can view and manage your API keys in the [Kiriku Dashboard](https://app.kiriku.app/dashboard/api-keys).

Your API keys carry many privileges, so be sure to keep them secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

## API Keys

Authentication to the API is performed via HTTP Basic Auth. Provide your API key as the basic auth username value. You do not need to provide a password.

<CodeGroup>
```bash cURL
curl https://api.kiriku.app/v1/users/me \
  -H "Authorization: Bearer sk_live_xxxxxxxxxxxxxxxx"
```

```javascript Node.js
const response = await fetch('https://api.kiriku.app/v1/users/me', {
  headers: {
    'Authorization': 'Bearer sk_live_xxxxxxxxxxxxxxxx'
  }
});
```
</CodeGroup>

If you need to authenticate via bearer auth (e.g., for a cross-origin request), use `-H "Authorization: Bearer sk_live_..."`.

## API Key Format

Kiriku API keys are prefixed with `sk_live_` or `sk_test_` depending on the environment.

| Environment | Prefix | Description |
| :--- | :--- | :--- |
| **Production** | `sk_live_` | Used for live data. Requests count towards your quota. |
| **Test** | `sk_test_` | Used for testing. Does not affect live data (coming soon). |
